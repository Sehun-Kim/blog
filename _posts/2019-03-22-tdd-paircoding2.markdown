# 주의 : 이 글은 TDD 설명서가 아니다!
이 글은 TDD 설명서도 아니고, 무조건적인 TDD 옹호글도 아니다. TDD는 개발방식의 하나일 뿐이고, 상황에따라 다른 개발방식이 필요할 것이다.

다만, 아무것도 몰랐던 코찔찔이 개발자가 좋은 멘토를 만나 새로운 개발방식에 눈을 뜬 후, 어떤 점이 좋았는지 이야기하고 싶어 작성한 **TDD 체험수기이다**.
